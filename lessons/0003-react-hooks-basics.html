Lesson 3 • React Hooks

Core React Hooks: State, Side Effects, and References

What are Hooks?

Before React 16.8, state and lifecycle methods were only available in Class components. Hooks allow functional components to use state and other React features.

If you know Vue's Composition API (e.g., ref, watchEffect), Hooks will feel familiar. However, because React re-runs the entire function on every render, Hooks rely on call order consistency.

The Core Hooks

1. useState

Declares state variables. It returns an array with the current state value and an updater function. 

const [count, setCount] = useState(0);

Asynchronous Updates

React state updates are batched and asynchronous. If you need to update state based on the previous state value, always pass an updater function: 

// Correct way for consecutive updates
setCount(prevCount => prevCount + 1);

2. useEffect

Handles side effects (data fetching, subscriptions, manual DOM mutations). It is the combined equivalent of Angular's lifecycle methods (ngOnInit, ngOnDestroy) or Vue's lifecycle hooks.

Its behavior is controlled by its dependency array:

  • No dependency array: Runs after every single render. (Rarely desired).
  • Empty dependency array []: Runs exactly once after the initial render (equivalent to ngOnInit).
  • With dependencies [dep1, dep2]: Runs after the initial render and then again whenever any listed dependency changes.

Effect Cleanup: If your effect returns a function, React runs this function when the component unmounts (equivalent to ngOnDestroy) or before running the effect again. 

useEffect(() => {
  const handleResize = () => console.log(window.innerWidth);
  window.addEventListener('resize', handleResize);

  // Cleanup function
  return () => {
    window.removeEventListener('resize', handleResize);
  };
}, []); // Runs once, cleans up on unmount

3. useRef

Creates a mutable object whose .current property persists across renders. Key characteristics:

  • Updating a ref does not trigger a re-render.
  • Used to store DOM node references (e.g., focusing an input).
  • Used to keep track of values across renders without triggering rendering (e.g., timer IDs).
const inputRef = useRef(null);
const focusInput = () => inputRef.current.focus();

return <input ref={inputRef} />;

The Rules of Hooks

React determines which state corresponds to which hook call by the order in which they are declared. Therefore, you must follow these rules:

  1. Only call Hooks at the top level: Do not call hooks inside loops, conditions, or nested functions.
  2. Only call Hooks from React functions: Call them from React functional components or custom hooks. Do not call them from standard JS functions.
