Lesson 8 • Testing

Testing React Applications

The React Testing Ecosystem

Testing React applications usually involves two core tools working together:

  • Test Runner (Vitest or Jest): Runs the tests, matches files (e.g., *.test.jsx), and provides assertion functions like expect(a).toBe(b). In modern Vite apps, **Vitest** is the standard.
  • Testing Library (React Testing Library - RTL): Provides helpers to render components in a mock DOM and query elements (e.g., finding buttons, typing into inputs) just like a real user would.

The React Testing Library (RTL) Philosophy

Key Rule of RTL

"The more your tests resemble the way your software is used, the more confidence they can give you."

Avoid testing internal component state or implementation details (e.g., do not verify if state.count === 1). Instead, test what the user sees: check if the text on the screen displays "1".

Basic Component Test Example

import { render, screen, fireEvent } from '@testing-library/react';
import { describe, it, expect } from 'vitest';
import Counter from './Counter';

describe('Counter Component', () => {
  it('should render initial count and increment on click', () => {
    // 1. Render the component in virtual browser environment
    render(<Counter />);

    // 2. Query elements by what the user sees (role, text)
    const button = screen.getByRole('button', { name: /click/i });
    const textElement = screen.getByText(/count: 0/i);

    expect(textElement).toBeInTheDocument();

    // 3. Simulate user interactions
    fireEvent.click(button);

    // 4. Assert updated state is drawn on the screen
    expect(screen.getByText(/count: 1/i)).toBeInTheDocument();
  });
});

Angular & Vue Parallels

How does this map to what you know?

  • Angular (TestBed): Angular uses TestBed to compile components, which often requires importing extensive lists of mock Modules, Providers, and Declarations. In React, there is no dependency injection or module system: you simply call render(<Component />) and mock external imports with standard JS mocks. It is significantly faster and cleaner.
  • Vue (Vue Test Utils): Conceptually identical. Vue also uses standard testing libraries (`@testing-library/vue`) to query and interact with components dynamically in a mock DOM.

High-Level E2E Testing Strategy

While unit tests verify component logic in isolation, **End-to-End (E2E) testing** runs your entire application in a real browser (Chrome, Firefox, Safari) and logs into a staging server to test full workflows.

The two standard E2E testing tools in modern development are **Playwright** and **Cypress**. For interviews, know that Playwright is highly favored for its speed, automatic waiting, and excellent support for parallel execution.
