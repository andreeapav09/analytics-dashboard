Memoization and Performance Optimization

The Core Problem: React's Render Behavior

By default, when a parent component's state changes in React, **the parent and ALL its children re-execute their functions**.

If your parent component displays a list of 10,000 metrics, and you type in a search input (updating state on every keystroke), React will re-run the render functions for the parent and all 10,000 child rows on every key press. Because JavaScript is single-threaded, this heavy calculations load blocks the main thread, resulting in laggy inputs and visible freeze frames.

To solve this when dealing with large datasets, React provides three core **Memoization** tools.

1. useMemo: Caching Expensive Computations

useMemo caches the *result* of a calculation between renders. It tells React: *"Do not run this sorting or filtering logic again unless the logs list or the search query has actually changed."*

Without useMemo (Slow / Runs on every render)

function Dashboard({ logs, searchQuery }) {
  // This filter loop runs on EVERY render (e.g., if a timer updates on the dashboard)
  const filteredLogs = logs.filter(log => log.message.includes(searchQuery));
  
  return <DataGrid logs={filteredLogs} />;
}

With useMemo (Optimized)

function Dashboard({ logs, searchQuery }) {
  // This only runs when logs or searchQuery changes. Other updates (like timers) skip it.
  const filteredLogs = useMemo(() => {
    return logs.filter(log => log.message.includes(searchQuery));
  }, [logs, searchQuery]); // Dependencies array
  
  return <DataGrid logs={filteredLogs} />;
}

Angular Parallel: Pure Pipes & Signal Computeds

In Angular, calling a method directly in your template binding (like <div>{{ filterData(data) }}</div>) is a major performance bottleneck because it runs on every change detection cycle.

To optimize this, Angular developers use **Pure Pipes** (which only re-run when their inputs change) or **computed Signals** (which cache results). useMemo is React's equivalent of a Pure Pipe or computed Signal.

2. React.memo: Caching Component Rendering

React.memo is a higher-order component. It checks the props of a child component. If the new props are identical to the previous props, React **skips rendering the child** completely, reusing its last rendered DOM node. 

// Wrap the component function in React.memo
const LogRow = React.memo(function LogRow({ log, onFlag }) {
  console.log("Rendering Row ID:", log.id);
  return (
    <div className="row">
      <span>{log.message}</span>
      <button onClick={() => onFlag(log.id)}>Flag</button>
    </div>
  );
});

Angular Parallel: ChangeDetectionStrategy.OnPush

By default, Angular change detection checks all components from top to bottom. When you set **`changeDetection: ChangeDetectionStrategy.OnPush`**, you tell Angular to skip checking this component and its children unless its `@Input()` references change.

React.memo does the exact same thing: it prevents React from traversing down into this component during a render pass unless the inputs (props) change.

3. useCallback: Stabilizing Callback References

In JavaScript, functions are objects. This means **referential equality** applies: `() => {} !== () => {}` (two identical functions created at different times do not point to the same memory address).

Every time the parent component re-renders, it redeclares and creates a new instance of any functions inside it. If you pass this function as a prop to a memoized child, the child sees a "changed prop" (because the function reference changed) and re-renders anyway, breaking React.memo.

useCallback caches the *function reference* itself, maintaining its memory pointer across renders.

The Bug (Recreates function, breaks child memoization)

function Dashboard() {
  const [theme, setTheme] = useState('dark');
  const handleFlag = (id) => dispatch({ type: 'FLAG', payload: id });

  // Every time theme changes, handleFlag is recreated.
  // This causes  to re-render, even if it is wrapped in React.memo!
  return <LogRow onFlag={handleFlag} />;
}

The Fix (Caches function reference)

function Dashboard() {
  const [theme, setTheme] = useState('dark');
  
  // React remembers this function reference. It never changes.
  const handleFlag = useCallback((id) => {
    dispatch({ type: 'FLAG', payload: id });
  }, []); // Empty dependencies = reference is cached forever
  
  // Now, changing theme will NOT cause LogRow to re-render!
  return <LogRow onFlag={handleFlag} />;
}
