Lazy Loading and Code Splitting

What is Code Splitting?

By default, bundlers like Webpack or Vite merge all your JavaScript code into a single, massive file (bundle). If your analytics app has heavy charts, CSV export libraries, and admin portals, the user has to download *everything* before the page loads.

Code Splitting splits your application into smaller JavaScript chunks that are downloaded on-demand (only when the user visits a certain tab or opens a heavy chart).

React.lazy and Suspense

React has built-in support for lazy loading components using React.lazy and the <Suspense> wrapper. 

import React, { lazy, Suspense } from 'react';

// 1. Lazy-load the heavy component using a dynamic import
const HeavyChart = lazy(() => import('./HeavyChart'));

function Dashboard() {
  return (
    <div>
      <h1>Analytics Dashboard</h1>

      {/* 2. Wrap the lazy component in Suspense with a loading fallback */}
      <Suspense fallback={<div>Loading Chart...</div>}>
        <HeavyChart />
      </Suspense>
    </div>
  );
}

Angular & Vue Parallels

How does this map to what you know?

  • Angular: Exactly like **Lazy-Loaded Routing** (using loadChildren) or Angular 17's new **@defer blocks**, which dynamically load parts of templates only when they enter the viewport or under custom trigger rules.
  • Vue: Exactly like Vue's **defineAsyncComponent** paired with Vue's own native **<Suspense>** tag.
