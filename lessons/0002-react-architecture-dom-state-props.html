React Architecture: Virtual DOM, State, and Props

Bridging the Gap: React vs. Angular vs. Vue

Since you have experience with Angular and Vue, understanding React's core rendering strategy is easiest when compared to what you already know:

  • Angular: Uses a dirty-checking mechanism (Zone.js) or fine-grained Signals to intercept data modifications and run change detection, direct DOM updates, and template updates.
  • Vue: Uses a reactive system (getter/setter interceptors in Vue 2 or Proxies in Vue 3) that tracks dependencies during render. When reactive state changes, Vue knows exactly which component needs to re-render.
  • React: Relies on an explicit, top-down execution model. When state changes in a component, React re-executes the entire component function (along with all its children, unless optimized) to generate a new Virtual DOM tree.

Real DOM vs. Virtual DOM

The Real DOM (Document Object Model) is the browser's internal tree representation of HTML. Modifying the real DOM directly is slow because updates trigger browser reflows (recalculating element positions and sizes) and repaints (drawing pixels on screen).

The Virtual DOM (VDOM) is a lightweight, in-memory JavaScript representation of the real DOM. When state changes:

  1. React runs the component functions to construct a new VDOM tree.
  2. React compares the new VDOM tree with the previous VDOM tree. This comparison process is called Reconciliation (powered by the React Fiber engine).
  3. React calculates the minimum number of changes required and updates only those specific nodes in the real DOM (called Commit phase).

State vs. Props

Data flow in React is strictly unidirectional (top-down). The two ways a component handles data are state and props:

State vs. Props Checklist

  • State: Internal, private data owned and managed entirely by the component itself. Can be modified by the component (using state updater functions). Modifying state triggers a re-render.
  • Props: External data passed down from a parent component. Props are read-only (immutable) to the child component. Changing props triggers a re-render.

comparative Example:

In Angular, you pass data into a child via @Input() and emit events via @Output(). In Vue, you use props and $emit.

In React, you pass both values and callback functions down as props: 

// Parent Component
function Parent() {
  const [count, setCount] = useState(0);
  return <Child counter={count} onIncrement={() => setCount(count + 1)} />;
}

// Child Component
function Child(props) {
  return (
    <div>
      <p>Count is: {props.counter}</p>
      <button onClick={props.onIncrement}>Increment</button>
    </div>
  );
}
