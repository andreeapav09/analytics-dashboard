Lesson 1 • Agentic Coding

Customizing AI Agents via Rules and Skills

The Paradigm of Agentic Coding

In traditional software development, coding assistants function as advanced autocompletes (e.g., inline suggestions). In Agentic Coding, the AI assistant acts as an autonomous collaborator. You provide a high-level goal, and the agent plans, researches, writes, edits, tests, and verifies the solution.

To make agentic coding reliable in professional environments, we use AI Coding Harnesses. A harness wraps the language model with execution sandboxes, local file tools, terminal access, and rulesets. It shifts the human's role from "writing code line-by-line" to "steering, reviewing, and defining constraints."

Rules vs. Skills: The Two Core Extensions

To customize an agent's behavior and system prompt, we define two types of files: Rules (using AGENTS.md or .md rule files) and Skills (using SKILL.md files inside skill directories).

Key Insight

Think of Rules as permanent, project-wide guardrails (e.g., "Always use TypeScript types"). Think of Skills as specialized, on-demand modules containing instructions, checklists, and scripts (e.g., "Deploying to production" or "Teaching a new framework").

1. Workspace & Global Rules (AGENTS.md)

Rules are broad instructions appended to the agent's baseline system prompt. In Antigravity, rules can be defined globally or scoped to a project workspace:

  • Global Rules: Applied to every workspace. Saved in the user's global config directory (e.g., C:\Users\z0043xbv\.gemini\config\AGENTS.md).
  • Project-Scoped Rules: Specific to the workspace. Saved in the project root under .agents/AGENTS.md.

Rules are excellent for defining code styling guidelines, test coverage thresholds, deprecated APIs to avoid, or architectural conventions.

2. Specialized Skills (SKILL.md)

Skills are self-contained directories that extend the agent with domain-specific knowledge and procedures. Each skill is located in a skills/<skill_name>/ directory under global or workspace config roots, containing:

  • SKILL.md: The entrypoint containing YAML frontmatter and detailed markdown instructions.
  • scripts/, examples/, resources/: Optional assets that support the skill's execution.

The YAML frontmatter defined in SKILL.md is critical. It must contain the skill's name and a clear description. The harness uses semantic matchers or keyword search on these fields to suggest or auto-load the skill when the user asks a matching question.

Typical Frontmatter Structure

Here is a typical SKILL.md configuration: 

---
name: react-component-generator
description: Generate standard React functional components with Storybook and RTL tests.
argument-hint: "Name of the component to generate (e.g., Button)"
---

## Component Guidelines
1. Always create the component file in `./src/components/`.
2. Generate a corresponding `.stories.tsx` storybook file.
3. Write a `.test.tsx` file asserting render and user event behaviors.
