Complex State Management with useReducer

What is useReducer?

As your React components grow in complexity, managing state with multiple useState hooks can make your code messy and hard to test.

useReducer is a built-in React hook designed to handle complex state logic. It is inspired by the Redux architecture: instead of updating state directly, you dispatch actions that describe what happened, and a central reducer function calculates the new state.

The Anatomy of useReducer

Using useReducer requires four key parts:

  • Initial State: The starting value of your state (often an object).
  • Action: A plain JavaScript object that describes what occurred. It must have a type property (e.g., { type: 'ADD_TODO', payload: 'Buy milk' }).
  • Reducer Function: A pure function that takes the current (state, action) and returns the *new* state. It contains no side effects.
  • Dispatch: A function provided by React that you call to send actions to the reducer.
const [state, dispatch] = useReducer(reducer, initialState);

Code Example: Managing a List of Logs

// 1. Initial State
const initialState = { logs: [] };

// 2. Reducer Function
function logReducer(state, action) {
  switch (action.type) {
    case 'ADD_LOG':
      return { logs: [...state.logs, action.payload] };
    case 'CLEAR_LOGS':
      return { logs: [] };
    default:
      return state;
  }
}

// 3. Component
function Logger() {
  const [state, dispatch] = useReducer(logReducer, initialState);

  return (
    <div>
      <button onClick={() => dispatch({ type: 'ADD_LOG', payload: 'Log entry' })}>
        Add Log
      </button>
      <button onClick={() => dispatch({ type: 'CLEAR_LOGS' })}>
        Clear Logs
      </button>
      <ul>
        {state.logs.map((log, index) => <li key={index}>{log}</li>)}
      </ul>
    </div>
  );
}

Angular & Vue Parallels

How does this map to what you know?

  • Angular (NgRx): If you have worked with NgRx or Redux in Angular, useReducer is conceptually identical. It uses the same unidirectional flow: **Component Dispatching Actions $\rightarrow$ Reducer pure function $\rightarrow$ New State**. The only difference is that useReducer is local to a component, whereas NgRx manages global state.
  • Vue (Pinia/Vuex): Vue's state changes are reactive (mutations directly edit properties). In contrast, React's reducer **must return a brand new object** representing the updated state (immutability).
