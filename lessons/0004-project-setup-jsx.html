Project Setup and JSX Fundamentals

Why Vite instead of Create React App?

In modern React development, we use Vite to initialize projects rather than the older, deprecated create-react-app (CRA).

  • Speed: Vite uses native ES Modules in the browser and compiles assets with Esbuild (written in Go), making development servers start almost instantly.
  • Flexibility: Vite is lightweight and highly configurable, unlike CRA which hides complex Webpack configs behind a rigid structure.

What is JSX?

JSX (JavaScript XML) is a syntax extension for JavaScript. It looks like HTML, but it compiles down to regular JavaScript function calls.

For example, when you write this JSX: 

const element = <h1 className="title">Hello World</h1>;

Under the hood, the compiler converts it to a standard JavaScript function call that returns a Virtual DOM node: 

// React 17+ Compiler Output
import { jsx as _jsx } from 'react/jsx-runtime';
const element = _jsx('h1', { className: 'title', children: 'Hello World' });

Because JSX is actually JavaScript, it gives you the full power of JavaScript loops, conditionals, and variables directly inside your markup.

The Strict Rules of JSX

Because JSX compiles to JavaScript objects, it has strict grammatical rules that differ from standard HTML:

  1. Return a Single Root Element: You cannot return multiple adjacent tags. They must be wrapped in a single parent tag (like a <div>) or a React Fragment (<>...</>).
  2. Close All Tags: Every tag must be explicitly closed. Self-closing tags must end with a slash: <img />, <br />, or <input />.
  3. camelCase Naming for Attributes: JSX attributes map directly to JavaScript DOM properties. Therefore, class is written as className, and inline styles are written as objects: style={{ color: 'red' }}.
