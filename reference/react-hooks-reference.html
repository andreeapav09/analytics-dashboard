Core React Hooks Cheat Sheet

Hook Reference Summary

  • useState: Persistent local component state. Updating triggers render.
  • useEffect: Handles side-effects, API calls, subscriptions, event listeners.
  • useRef: Persistent reference value. Updating does not trigger render. Accesses DOM nodes directly.

The Dependency Array Rulebook

Syntax Trigger Frequency Ideal Use Cases
useEffect(fn) After EVERY render Rare. Debugging, global performance sync.
useEffect(fn, []) Exactly ONCE (on mount) Data fetching, setup listeners, one-time logs.
useEffect(fn, [val]) On mount, and whenever val changes Re-fetching data on ID change, updating calculations.

Crucial Rules to Remember

  • Rule 1: Call Hooks only at the top level. Avoid nested functions, conditions (if blocks), or loops.
  • Rule 2: Call Hooks only from functional components or custom hooks. Standard JS modules cannot parse hooks.
  • Rule 3: Always clean up listeners, intervals, and subscriptions inside the return function of useEffect to prevent memory leaks.
